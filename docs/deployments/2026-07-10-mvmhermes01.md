## Findings (deviations from the docs)

1. **DNS is mangled on mvmhermes01** — `suite-db.example.ad.example.com` does
   not resolve. All DSNs use the IP `192.168.1.22` instead. The operator
   should fix DNS or add `/etc/hosts` entries.

2. **agent-notes has two databases** — the native DB (`agent_notes` database,
   owned by `notes_service`) is separate from the regista store (`regista`
   database, owned by `regista_service`). The suite.env must set both
   `REGISTA_DSN` and `AGENT_NOTES_DSN`. The agent-notes `resolve_dsn()`
   function does not fall back to `REGISTA_DSN` — it only reads
   `AGENT_NOTES_DSN` or the config file. This is a Plan 017 tail item.

3. **agent-notes requires `AGENT_NOTES_REGISTA_WRITES=1`** to enable
   writes. This env var is not documented in the suite.env.example. The
   agent-notes config.json also needs to be created with both DSNs.

4. **regista's doctor does not emit a top-level `ok` field** — the umbrella
   was inferring `false`. Fixed in the umbrella by inferring `ok` from
   checks when the field is absent. A proper fix is to add the `ok` field
   to regista's doctor output (same issue as acb Plan 006).

5. **Pre-existing data issue: `links_audit` failure** — 11 dangling
   memory-to links in the `agent_notes` database. This is NOT caused by
   this deployment; it exists in the shared store. Fixing it requires a
   data cleanup query. Until fixed, `suite_ok` will be false.

6. **`content_encryption` warning in cairn** — content encryption is ON but
   no content key is configured (`CAIRN_CONTENT_KEY_REF` /
   `CAIRN_CONTENT_KEY_PATH`). Content capture will store plaintext until a
   key is set. This is expected for a first deployment.

7. **PyNaCl (`[ed25519]` extra) must be explicitly installed** for regista,
   agent-notes, and cairn. `uv tool install` does not install extras by
   default. The fix: `uv tool install --with pynacl <package>` or
   `uv tool install --with "regista[ed25519]" <package>`.

8. **Hermes service is inactive** — `systemctl is-active hermes` returns
   `inactive`. The cairn plugin is installed but Hermes is not running.
   The operator should start Hermes to activate the attestation integration.
